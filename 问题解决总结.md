# 文件上传问题 - 完整解决方案

## 📋 问题诊断

### 原始问题
```
[文件上传] 上传失败
uploadUrl: "http://192.168.111.1:8080/api/files/upload"
useCloud: true
Error: 文件上传失败，请检查网络连接
```

**矛盾点：** `useCloud: true` 但使用了本地地址

### 根本原因
1. **环境检测逻辑问题（已修复 ✅）** - `getUploadUrl()` 没有正确切换到云托管地址
2. **云托管缺少接口（待部署 ⏳）** - 云托管运行旧版本代码，没有 `/api/files/upload` 接口

## 🐛 常见问题

### Q1: 本地上传失败，提示网络错误

**可能原因：**
- 后端未启动
- 不在同一 WiFi
- 防火墙阻止
- IP 地址错误

**解决步骤：**
1. 运行 `检查上传配置.cmd`
2. 确认后端运行中
3. 手机浏览器访问 `http://IP:8080/api/idols`
4. 检查 IP 地址是否正确

---

### Q2: 云托管上传返回 503/504

**原因：** 云托管服务未启动或启动失败

**解决：**
1. 云托管控制台 → 查看实例状态
2. 查看实例日志 → 检查启动错误
3. 确认环境变量配置正确
4. 重新部署或重启实例

---

### Q3: 云托管上传返回 404

**原因：** 云托管运行的是旧版本代码

**解决：** 部署最新代码到云托管

---

### Q4: 上传成功但图片不显示

**可能原因：**
- 图片路径拼接错误
- 文件实际未保存
- 静态文件访问配置问题

**检查：**
1. 查看上传返回的 URL
2. 浏览器直接访问图片 URL
3. 检查服务器 `uploads` 目录
