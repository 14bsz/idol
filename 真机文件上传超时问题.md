# 真机预览文件上传超时问题

## 🔍 错误信息

```
uploadFile:fail upload fail:-118:net::ERR_CONNECTION_TIMED_OUT
```

**翻译：** 文件上传失败，连接超时

## 🎯 问题原因

### 环境判断不一致

真机预览时的环境配置：

| 功能 | 当前配置 | 结果 |
|------|---------|------|
| **API 请求** | 使用云托管 (`wx.cloud.callContainer`) | ✅ 正常 |
| **文件上传** | 使用本地地址 (`http://192.168.111.1:8080`) | ❌ 超时 |

### 根本原因

```javascript
// env.js 中的配置
const UPLOAD_URLS = {
  develop: 'http://192.168.111.1:8080/api/files/upload',  // ← 真机无法访问
  trial: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload',
  release: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload'
};

// 原来的逻辑
const getUploadUrl = (envVersion) => {
  const currentEnv = envVersion || getRuntimeEnvVersion();
  return UPLOAD_URLS[currentEnv] || UPLOAD_URLS.develop;
  // ↑ 真机预览时 envVersion = 'develop'，返回本地地址
};
```

**问题：** 手机无法访问电脑的 `192.168.111.1:8080`（除非在同一局域网）

---

## ✅ 解决方案（已修复）

### 修改 1：优化 getUploadUrl 逻辑

**文件：** `miniprogram/utils/env.js`

```javascript
const getUploadUrl = (envVersion) => {
  const currentEnv = envVersion || getRuntimeEnvVersion();
  
  // 🎯 真机预览时使用云托管地址
  // 因为手机无法访问本机的 localhost 或局域网地址
  if (currentEnv === 'develop' && useCloudContainer(envVersion)) {
    return UPLOAD_URLS.trial; // 使用云托管的上传地址
  }
  
  return UPLOAD_URLS[currentEnv] || UPLOAD_URLS.develop;
};
```

**逻辑说明：**

```
真机预览环境判断：
├─ envVersion = 'develop'（开发环境）
├─ useCloudContainer() = true（真机无法访问本地）
└─ 返回：云托管上传地址 ✅

开发者工具环境判断：
├─ envVersion = 'develop'（开发环境）
├─ useCloudContainer() = false（工具可以访问本地）
└─ 返回：本地上传地址 ✅
```

### 修改 2：增强上传日志

**文件：** `miniprogram/app.js`

```javascript

}
```

## ⚠️ 注意事项

### 1. 云托管服务必须运行

真机预览时依赖云托管服务，请确保：
- ✅ 云托管服务状态为"运行中"
- ✅ 后端代码已部署到云托管
- ✅ 文件上传接口 `/api/files/upload` 可用

### 2. 合法域名配置

小程序需要配置云托管域名：

**微信公众平台 → 开发管理 → 开发设置 → 服务器域名**

```
uploadFile合法域名：
https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com
```

### 3. 文件大小限制

检查配置：
- 小程序端：默认 10MB
- 后端 Spring Boot：`application.yml` 中配置

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB
```

### 4. 使用本地上传（可选）

如果想在真机预览时也使用本地后端：

**前提条件：**
- 手机和电脑在同一 WiFi
- 电脑防火墙允许 8080 端口访问

**修改方法：**

在 `env.js` 中注释掉云托管判断：

```javascript
const useCloudContainer = (envVersion) => {
  // 强制使用本地后端
  return false;
  
  // 原来的逻辑...
};
```

## 🐛 其他可能的错误

### 错误 1：401 Unauthorized

```
上传失败: 登录已过期
```

**原因：** Token 过期
**解决：** 重新登录

---

### 错误 2：413 Payload Too Large

```
上传失败: 文件太大
```

**原因：** 文件超过限制
**解决：** 
1. 压缩图片
2. 增大后端配置限制

---

### 错误 3：500 Internal Server Error

```
上传失败: 服务器错误
```

**原因：** 后端处理失败
**解决：** 
1. 查看云托管日志
2. 检查后端文件存储配置
3. 检查磁盘空间
