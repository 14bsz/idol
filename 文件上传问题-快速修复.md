# 文件上传问题 - 快速修复

## 问题症状

```
[文件上传] 上传失败
uploadUrl: "http://192.168.111.1:8080/api/files/upload"
useCloud: true
Error: 文件上传失败，请检查网络连接
```

**矛盾点**：`useCloud: true` 但 `uploadUrl` 仍是本地地址

## 已执行的修复

### 1. 添加调试日志

在 `miniprogram/utils/env.js` 中添加了详细的调试日志，可以追踪：
- 环境版本检测
- 云托管模式判断
- 上传地址选择逻辑

### 2. 增强 `useCloudContainer()` 函数

```javascript
const useCloudContainer = (envVersion) => {
  const currentEnv = envVersion || getRuntimeEnvVersion();
  if (currentEnv !== 'develop') {
    console.log('[env.useCloudContainer] 非开发环境，使用云托管');
    return true;
  }

  try {
    const systemInfo = wx.getSystemInfoSync();
    const isDevTools = systemInfo.platform === 'devtools';
    console.log('[env.useCloudContainer] platform:', systemInfo.platform, 'isDevTools:', isDevTools);
    
    return !isDevTools;  // 真机返回 true
  } catch (e) {
    console.error('[env.useCloudContainer] 获取系统信息失败:', e);
    return false;
  }
};
```

### 3. 增强 `getUploadUrl()` 函数

```javascript
const getUploadUrl = (envVersion) => {
  const currentEnv = envVersion || getRuntimeEnvVersion();
  const isCloudMode = useCloudContainer(envVersion);
  
  console.log('[env.getUploadUrl] currentEnv:', currentEnv, 'isCloudMode:', isCloudMode, 'USE_LOCAL:', USE_LOCAL_UPLOAD_IN_DEVICE);
  
  if (currentEnv === 'develop' && isCloudMode) {
    if (USE_LOCAL_UPLOAD_IN_DEVICE) {
      console.log('[env.getUploadUrl] 使用本地上传地址');
      return UPLOAD_URLS.develop;
    }
    console.log('[env.getUploadUrl] 使用云托管上传地址');
    return UPLOAD_URLS.trial;  // 返回云托管地址
  }
  
  const finalUrl = UPLOAD_URLS[currentEnv] || UPLOAD_URLS.develop;
  console.log('[env.getUploadUrl] 最终上传地址:', finalUrl);
  return finalUrl;
};
```

## 可能的问题场景

### 场景 1：云托管服务未启动

**症状**：上传地址正确，但返回 502 / 503 错误

**解决**：
```bash
# 检查云托管服务状态
# 前往腾讯云控制台 → 云托管 → 查看服务状态
```

### 场景 2：Authorization Token 无效

**症状**：返回 401 Unauthorized

**解决**：
1. 清除小程序缓存
2. 重新登录
3. 确认后端 Token 验证逻辑

### 场景 3：文件大小超限

**症状**：上传大文件时失败

**解决**：
检查后端配置 `application.yml`:
```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB      # 调整文件大小限制
      max-request-size: 10MB
```

### 场景 4：CORS 跨域问题

**症状**：浏览器报 CORS 错误（小程序不会有此问题）

**解决**：
检查 `CorsConfig.java` 配置

## 如果问题依然存在

### 选项 A：强制使用云托管（临时方案）

修改 `miniprogram/utils/env.js`：

```javascript
const useCloudContainer = (envVersion) => {
  // 临时强制使用云托管
  return true;
};
```

### 选项 B：使用本地调试（手机和电脑同一 WiFi）

1. 获取电脑 IP 地址：
```cmd
ipconfig
```

2. 更新 `miniprogram/utils/env.js`：
```javascript
const API_BASE_URLS = {
  develop: 'http://192.168.1.XXX:8080/api',  // 替换为你的 IP
  trial: '',
  release: ''
};

const UPLOAD_URLS = {
  develop: 'http://192.168.1.XXX:8080/api/files/upload',  // 替换为你的 IP
  trial: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload',
  release: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload'
};

const USE_LOCAL_UPLOAD_IN_DEVICE = true;  // 启用本地上传
```

3. 启动后端：
```cmd
cd d:\SOFT\爱豆时光日记\backend
mvn spring-boot:run
```

4. 配置防火墙允许 8080 端口

### 选项 C：使用开发者工具本地调试

在开发者工具中：
- 不会触发云托管切换
- 直接使用 `UPLOAD_URLS.develop`
- 需要后端在本地运行
