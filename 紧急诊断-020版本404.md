# 🚨 紧急诊断：版本 020 仍然返回 404

## 当前状态

- ✅ 只保留了版本 020
- ✅ 版本 020 有 1 个实例运行
- ✅ 其他 API 正常（idols, diaries 等返回 200）
- ❌ 文件上传接口返回 404

## 测试结果

```bash
curl -X POST https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload
```

返回：
```json
{
  "timestamp": "2026-06-08T07:44:59.251+00:00",
  "status": 404,
  "error": "Not Found",
  "path": "/api/files/upload"
}
```

## 可能的原因

### 原因 1：版本 020 部署的 JAR 包不是最新的 ⚠️

**症状：** 
- 本地 `cloud-deploy/app.jar` 包含 FileController
- 但云端部署的可能是旧的 JAR

**验证方法：**
查看版本 020 的实例日志，搜索：
```
FileController
```

如果找不到，说明部署的 JAR 包不对。

### 原因 2：FileController 没有被 Spring 扫描到

**症状：**
- JAR 包有 FileController.class
- 但 Spring 启动时没有扫描到这个类

**验证方法：**
查看版本 020 的实例日志，搜索：
```
Mapped "{[/api/files/upload]
```

或者搜索：
```
RequestMappingHandlerMapping
```

看看有没有 `/api/files/upload` 的映射。

### 原因 3：@RestController 或 @RequestMapping 注解丢失

**症状：**
- FileController 类被扫描到
- 但没有注册为 Controller

**验证方法：**
检查 FileController.java 的注解是否完整。
