# 部署置顶功能到云托管环境

## 问题确认

通过真机预览的日志分析，确认问题：

```
[云托管响应] statusCode: 404 data: {"path":"/api/diaries/1/pin"}
```

**根本原因：云托管环境的后端代码是旧版本，缺少置顶接口 `PUT /api/diaries/{id}/pin`**

- ✅ 本地开发环境：有置顶功能（开发者工具正常）
- ❌ 云托管环境：没有置顶功能（真机预览失败）

## 需要部署的内容

### 1. 数据库迁移
云托管数据库需要执行：`backend/src/main/resources/migration_add_pinned.sql`

### 2. 后端代码
需要包含以下更新：
- `DiaryController.java` - 新增 `togglePinDiary` 方法
- `DiaryService.java` - 新增按 pinned 排序的逻辑
- `Diary.java` - 新增 `pinned` 字段

### 3. 前端代码
小程序已经包含置顶功能，无需额外部署。

## 部署步骤

### 步骤 1：连接云托管数据库

有两种方式：

**方式 A：通过微信云开发控制台**
1. 登录 [微信云开发控制台](https://console.cloud.tencent.com/tcb)
2. 选择环境：`prod-d7gv8c42oa25aafb0`
3. 进入 "云托管" → "数据库"
4. 使用 SQL 编辑器执行迁移脚本

**方式 B：通过 MySQL 客户端**
1. 获取云数据库连接信息（在云托管控制台查看）
2. 使用 MySQL Workbench 或命令行连接
3. 执行迁移脚本

### 步骤 2：执行数据库迁移

执行 `migration_add_pinned.sql` 文件内容：

```sql
USE idol_diary;

-- 添加置顶字段
ALTER TABLE `diary` 
ADD COLUMN `pinned` TINYINT NOT NULL DEFAULT 0 COMMENT '是否置顶 0-否 1-是' AFTER `tags`;

-- 添加索引以优化置顶日记的查询
ALTER TABLE `diary` 
ADD INDEX `idx_pinned_created_at` (`pinned`, `created_at` DESC);
```

**验证迁移成功：**
```sql
DESC diary;  -- 查看表结构，应该包含 pinned 字段
SHOW INDEX FROM diary;  -- 查看索引，应该包含 idx_pinned_created_at
```

### 步骤 3：构建后端项目

在 `backend` 目录下执行：

```bash
# Windows CMD
mvn clean package -DskipTests

# 或 Windows PowerShell
mvn clean package -DskipTests
```

这将生成 `backend/target/idol-diary-backend-0.0.1-SNAPSHOT.jar`

### 步骤 4：部署到云托管

**方式 A：通过微信开发者工具（推荐）**

1. 打开微信开发者工具
2. 点击顶部菜单 "云开发" → "云托管"
3. 选择服务：`springboot-h9k1`
4. 点击 "上传代码"
5. 选择 `backend` 目录
6. 配置构建参数（如果需要）
7. 点击 "部署"

**方式 B：通过腾讯云控制台**

1. 登录 [腾讯云微信云托管控制台](https://console.cloud.tencent.com/tcb)
2. 选择环境：`prod-d7gv8c42oa25aafb0`
3. 进入 "云托管" → 选择服务 `springboot-h9k1`
4. 点击 "版本管理" → "新建版本"
5. 上传 JAR 包或配置镜像
6. 设置环境变量（如果有变化）
7. 点击 "部署"

**方式 C：使用 Dockerfile**

如果项目使用 Docker 部署：

```bash
# 在 backend 目录下
docker build -t idol-diary-backend:latest .

# 推送到云托管的镜像仓库
docker tag idol-diary-backend:latest [镜像仓库地址]/idol-diary-backend:latest
docker push [镜像仓库地址]/idol-diary-backend:latest
```

然后在云托管控制台选择新镜像版本部署。

### 步骤 5：验证部署

部署完成后，使用以下方式验证：

通过云托管公网地址测试：

## 注意事项

### 1. 数据库备份
在执行迁移前，建议先备份数据库：
```sql
-- 备份 diary 表
CREATE TABLE diary_backup AS SELECT * FROM diary;
```

### 2. 环境变量检查
确认云托管环境变量配置正确：
- `MYSQL_ADDRESS` - 数据库地址
- `MYSQL_USERNAME` - 数据库用户名
- `MYSQL_PASSWORD` - 数据库密码

### 3. 灰度发布（可选）
如果担心影响线上用户，可以使用云托管的灰度发布功能：
1. 创建新版本，设置流量比例为 10%
2. 观察日志和监控指标
3. 逐步提升流量比例到 100%

### 4. 回滚准备
记录当前版本号，以便出现问题时快速回滚：
- 在云托管控制台可以查看历史版本
- 点击"回滚"可以快速恢复到之前的版本

## 常见问题

### Q1: 数据库迁移失败，提示字段已存在
**原因：** 可能之前已经执行过迁移
**解决：** 检查字段是否存在：
```sql
DESC diary;
```
如果 `pinned` 字段已存在，跳过数据库迁移步骤。

### Q2: 部署后仍然 404
**可能原因：**
1. 新版本未启动成功（检查云托管日志）
2. 流量未切换到新版本（检查版本流量配置）
3. 容器启动失败（检查构建日志）

**排查方法：**
1. 在云托管控制台查看"实例日志"
2. 检查 Spring Boot 启动日志
3. 确认接口注册成功：`Mapped "{[/api/diaries/{id}/pin],methods=[PUT]}"`

### Q3: 其他接口也报错（如 collections）
从日志中发现：
```
Unknown column 'tags' in 'field list' ... FROM collection
```
这说明 `collection` 表也缺少 `tags` 字段。需要同时修复这个问题。

## 其他发现的问题

### Collection 表缺少 tags 字段

日志显示：
```
Unknown column 'tags' in 'field list' ... FROM collection
```

需要额外执行：
```sql
ALTER TABLE `collection` 
ADD COLUMN `tags` VARCHAR(500) COMMENT '标签，JSON 数组格式' AFTER `notes`;
```
