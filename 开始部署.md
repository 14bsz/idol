## 🎯 快速部署（3 步完成）

### 第 1 步：打开微信开发者工具

1. 打开你的小程序项目
2. 点击顶部菜单 **"云开发"** 按钮
3. 选择 **"云托管"** 标签

### 第 2 步：上传代码

1. 选择服务：**springboot-h9k1**
2. 点击 **"新建版本"** 或 **"上传代码"**
3. 选择整个 **`cloud-deploy`** 文件夹上传
4. 配置部署参数：
   ```
   端口：8080
   CPU：0.5 核（推荐）
   内存：1 GB（推荐）
   实例数：1
   ```
5. 点击 **"确定"** 开始部署

### 第 3 步：等待完成

1. 查看构建进度（约 5-10 分钟）
2. 等待实例状态变为 **"运行中"** ✅
3. 点击 **"日志"** 查看启动信息
4. 看到 `Started IdolDiaryApplication` 表示成功 🎉

## 🆘 常见问题

### Q1: 找不到云托管入口？

**解决：**
1. 确认小程序已开通云开发
2. 微信开发者工具 → 顶部菜单栏 → "云开发"
3. 或右上角工具栏中的云开发图标

### Q2: 服务列表中没有 springboot-h9k1？

**解决：**
1. 检查是否选择了正确的云环境
2. 如果服务不存在，需要先创建服务
3. 服务名可以自定义，记得更新小程序配置中的 `CLOUD_SERVICE_NAME`

### Q3: 部署失败，提示构建超时？

**解决：**
1. 检查 Dockerfile 配置
2. 本部署包使用预构建 JAR，应该很快（< 5 分钟）
3. 检查网络连接
4. 重试部署

### Q4: 实例启动失败？

**可能原因：**
- 数据库连接失败
- 环境变量未配置
- 内存不足

**解决：**
1. 查看实例日志中的错误信息
2. 检查环境变量配置
3. 尝试增加内存到 2GB

### Q5: 接口返回 404？

**检查：**
1. 部署的是否是最新的 JAR 包（2026-06-08 构建）
2. 查看实例日志，确认应用启动成功
3. 测试其他接口（如 /api/idols）是否正常
