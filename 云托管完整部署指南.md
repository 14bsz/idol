# 云托管完整部署指南

## 🚨 当前问题

真机预览时发现云托管环境缺少以下接口：

| 接口 | 状态 | 影响功能 |
|------|------|---------|
| `PUT /api/diaries/{id}/pin` | ❌ 404 | 日记置顶 |
| `POST /api/files/upload` | ❌ 404/503 | 文件上传（头像、壁纸、收藏图片） |

**根本原因：** 云托管环境运行的是旧版本代码

---

## ✅ 解决方案

### 方案 A：立即部署到云托管（推荐）

**优点：** 
- 一劳永逸
- 真机预览和正式版都能用
- 完整功能支持

**步骤：** 参考下方"部署步骤"

---

### 方案 B：临时使用本地后端

**前提条件：**
- ✅ 手机和电脑连接同一 WiFi
- ✅ 电脑防火墙允许 8080 端口

**配置方法：**

1. **确认手机和电脑在同一 WiFi**

2. **修改 `miniprogram/utils/env.js`**（已修改）
   ```javascript
   // 设置为 true 使用本地上传
   const USE_LOCAL_UPLOAD_IN_DEVICE = true;
   ```

3. **启动本地后端**
   ```bash
   cd backend
   mvn spring-boot:run
   ```

4. **真机预览测试**

**优点：** 
- 无需等待部署
- 开发调试快速

**缺点：**
- 依赖本地服务器运行
- 需要同一网络
- 体验版/正式版仍然不可用

### 步骤 3：部署到云托管

#### 方式 1：通过微信开发者工具（推荐）

1. **打开微信开发者工具**

2. **进入云托管管理**
   - 顶部菜单 → 云开发 → 云托管

3. **选择服务**
   - 服务名：`springboot-h9k1`

4. **上传代码**
   - 方法 A：直接上传整个 `backend` 目录
   - 方法 B：只上传 `backend/target/*.jar` 文件

5. **配置部署参数**
   ```
   端口：8080
   CPU：0.5 核（可按需调整）
   内存：1GB（可按需调整）
   副本数：1
   ```

6. **确认环境变量**
   ```
   MYSQL_ADDRESS=<云数据库地址>
   MYSQL_USERNAME=<数据库用户名>
   MYSQL_PASSWORD=<数据库密码>
   ```

7. **点击"部署"**

8. **等待部署完成**
   - 通常需要 3-10 分钟
   - 查看"实例日志"确认启动成功

**云托管控制台：**
- 实例日志 → 选择实例 → 实时日志

**关注：**
- 启动日志：确认服务启动成功

#### 常见错误日志

**错误 1：数据库连接失败**
```
Could not open JDBC Connection
```
**解决：** 检查环境变量配置

---

**错误 2：文件上传权限问题**
```
java.nio.file.AccessDeniedException: uploads/2026/06/08/xxx.jpg
```
**解决：** 检查文件目录权限，或修改 `application.yml` 中的 `file.upload.path`

---

**错误 3：内存不足**
```
java.lang.OutOfMemoryError: Java heap space
```
**解决：** 增加实例内存配置

## 🔄 回滚方案

如果部署后出现严重问题：

### 回滚后端

**云托管控制台：**
1. 版本管理
2. 选择上一个稳定版本
3. 点击"回滚"
4. 流量自动切换到旧版本

### 回滚数据库（谨慎）

```sql
-- 如果需要移除新添加的字段
ALTER TABLE `diary` DROP COLUMN `pinned`;
ALTER TABLE `diary` DROP INDEX `idx_pinned_created_at`;
ALTER TABLE `collection` DROP COLUMN `tags`;
```

**警告：** 回滚数据库会导致数据丢失，请先备份！

## ⚠️ 注意事项

### 1. 部署时间

- **非高峰时段部署**：避免影响用户
- **预计时间**：5-15 分钟
- **灰度发布**：先 10% 流量测试，再逐步提升

### 2. 环境变量

确认以下环境变量正确配置：
```
MYSQL_ADDRESS=xxx.xxx.xxx.xxx:3306
MYSQL_USERNAME=root
MYSQL_PASSWORD=xxxxxx
```

### 3. 文件存储

**默认配置：** `file.upload.path=uploads`
- 文件存储在容器内的 `uploads` 目录
- 容器重启后文件**可能丢失**

**推荐方案（生产环境）：**
- 使用对象存储（COS）
- 或配置持久化卷

### 4. 资源配置

**开发阶段：**
- CPU: 0.25-0.5 核
- 内存: 512MB-1GB
- 实例数: 1

**生产环境：**
- CPU: 1-2 核
- 内存: 2-4GB
- 实例数: 2-3（高可用）
