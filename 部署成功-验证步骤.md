### 使用 curl 测试

**⚠️ 需要先获取有效的 Authorization Token**

```bash
curl -X POST "https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload" \
  -H "Authorization: YOUR_TOKEN" \
  -F "file=@test.jpg"
```

**预期返回（成功）：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "url": "/uploads/2026/06/08/xxxx-xxxx-xxxx.jpg",
    "filename": "xxxx-xxxx-xxxx.jpg"

```

✅ **返回 200** = 文件上传功能正常！🎉

**其他可能的返回：**
- `401 Unauthorized` = Token 无效或未提供（需要重新获取 Token）
- `404 Not Found` = 接口不存在（说明还是部署了旧代码）
- `500 Internal Server Error` = 服务器错误（查看日志）

## ⚠️ 常见问题

### Q1: 实例状态一直是「启动中」？

**原因：** 应用启动需要时间（通常 1-2 分钟）

**解决：**
- 等待 2-3 分钟
- 查看实例日志
- 如果超过 5 分钟还在启动，查看日志中的错误

---

### Q2: 启动日志中有数据库连接错误？

**错误示例：**
```
Could not open JDBC Connection
Access denied for user 'root'@'xxx' (using password: YES)
```

**原因：** 环境变量未配置或配置错误

**解决：**
1. 云托管控制台 → 服务设置 → 环境变量
2. 检查以下环境变量：
   ```
   MYSQL_ADDRESS=<云数据库地址>:3306
   MYSQL_USERNAME=<数据库用户名>
   MYSQL_PASSWORD=<数据库密码>
   ```
3. 修改后需要重启实例

---

### Q3: 文件上传返回 401？

**原因：** Authorization Token 无效或未提供

**解决：**
1. 重新登录小程序获取新 Token
2. 参考文档：`如何获取Authorization-Token.md`
3. 在测试时使用正确的 Token

---

### Q4: 文件上传返回 404？

**原因：** 接口不存在（可能部署了旧代码）

**解决：**
1. 检查部署的版本号：springboot-h9k1-020
2. 查看构建日志，确认使用了 Java 17
3. 如果确认是最新版本，查看应用日志

---

### Q5: 上传成功但图片不显示？

**原因：** 图片路径问题

**检查：**
1. 上传返回的 URL：`/uploads/2026/06/08/xxx.jpg`
2. 完整 URL：`https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/uploads/2026/06/08/xxx.jpg`
3. 浏览器直接访问完整 URL 测试
