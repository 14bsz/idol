## 临时方案：使用本地后端上传

**已自动配置：** `USE_LOCAL_UPLOAD_IN_DEVICE = true`

### 前提条件
1. ✅ 手机和电脑连接**同一 WiFi**
2. ✅ 电脑上的后端服务正在运行
3. ✅ IP 地址配置正确：`192.168.111.1`

### 第一步：确认 IP 地址

打开命令提示符，运行：
```cmd
ipconfig
```

找到你的 WiFi IP 地址，例如：
```
无线局域网适配器 WLAN:
   IPv4 地址 . . . . . . . . . . . . : 192.168.111.1
```

**如果 IP 不是 `192.168.111.1`**，需要更新配置：

打开 `miniprogram/utils/env.js`，修改：
```javascript
const API_BASE_URLS = {
  develop: 'http://你的实际IP:8080/api',  // 例如 192.168.1.100
  trial: '',
  release: ''
};

const UPLOAD_URLS = {
  develop: 'http://你的实际IP:8080/api/files/upload',  // 例如 192.168.1.100
  trial: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload',
  release: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload'
};
```

### 第二步：启动后端服务

打开新的命令提示符窗口：

```cmd
cd d:\SOFT\爱豆时光日记\backend
mvn spring-boot:run
```

**或者**如果已经打包过：
```cmd
cd d:\SOFT\爱豆时光日记\backend
java -jar target\idol-diary-backend-0.0.1-SNAPSHOT.jar
```

**等待看到启动成功日志：**
```
Started IdolDiaryApplication in X.XXX seconds
Tomcat started on port(s): 8080 (http)
```

### 第三步：配置防火墙

允许 8080 端口入站连接（以管理员身份运行 PowerShell）：

```powershell
netsh advfirewall firewall add rule name="Java Backend 8080" dir=in action=allow protocol=TCP localport=8080
```

### 第四步：测试连接

在手机浏览器访问：
```
http://192.168.111.1:8080/api/idols
```

如果能看到 JSON 数据或者登录提示，说明连接成功 ✅

### 第五步：重新编译小程序

1. 保存所有修改的文件
2. 微信开发者工具会自动重新编译
3. 查看控制台日志：

```
[env.getUploadUrl] currentEnv: develop isCloudMode: true USE_LOCAL: true
[env.getUploadUrl] 使用本地上传地址
[App] API配置完成: {
  uploadUrl: "http://192.168.111.1:8080/api/files/upload"
}
```

✅ 确认 `uploadUrl` 是本地地址

### 第六步：测试上传

1. 真机预览
2. 选择图片上传
3. 查看日志：

```
[文件上传] 开始上传 {uploadUrl: "http://192.168.111.1:8080/api/files/upload"}
[文件上传] 上传成功 {statusCode: 200}
```

## 常见问题

### Q1: 手机无法访问电脑后端

**可能原因：**
1. 不在同一 WiFi
2. 防火墙阻止连接
3. IP 地址错误

**检查步骤：**
1. 确认手机和电脑连接同一 WiFi
2. 手机浏览器访问 `http://你的IP:8080/api/idols`
3. 如果仍然无法访问，尝试关闭防火墙测试

### Q2: 上传成功但图片不显示

**可能原因：** 图片路径问题

**检查：**
1. 查看上传返回的 URL：`/uploads/2026/06/08/xxx.jpg`
2. 确认 `resolveMediaUrl` 函数正确拼接路径
3. 手机浏览器访问完整图片 URL 测试

### Q3: 后端报错 "Access Denied"

**解决：** 检查 `uploads` 目录权限
```cmd
# 在 backend 目录创建 uploads 文件夹
cd d:\SOFT\爱豆时光日记\backend
mkdir uploads
```

## 最终解决方案

**这只是临时方案**，最终还是需要部署到云托管：

### 部署步骤概要

1. **构建后端**
   ```cmd
   cd backend
   mvn clean package -DskipTests
   ```

2. **上传到云托管**
   - 微信开发者工具 → 云开发 → 云托管
   - 上传 `backend` 目录或 JAR 包

3. **等待部署完成**
   - 查看实例日志确认启动成功

4. **测试云托管接口**
   ```bash
   curl https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload
   ```

5. **恢复云托管上传配置**
   ```javascript
   const USE_LOCAL_UPLOAD_IN_DEVICE = false;
   ```

## 优缺点对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **本地上传（当前）** | 快速测试<br>无需等待部署 | 需要同一网络<br>依赖本地服务<br>体验版不可用 |
| **云托管上传** | 稳定可靠<br>随时可用<br>体验版可用 | 需要部署<br>需要等待时间 |
