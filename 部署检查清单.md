## 常见问题快速排查

### 问题 1：数据库迁移执行失败

**错误：** `Table 'idol_diary' doesn't exist`
**原因：** 数据库名称不正确
**解决：** 检查云托管数据库的实际名称，修改 SQL 中的 `USE idol_diary;`

---

### 问题 2：后端构建失败

**错误：** `Failed to execute goal org.apache.maven.plugins:maven-compiler-plugin`
**原因：** JDK 版本不匹配或依赖下载失败
**解决：** 
1. 检查 JDK 版本（项目需要 Java 8 或更高）
2. 清理缓存：`mvn clean`
3. 重新构建：`mvn package -DskipTests`

---

### 问题 3：部署后仍然 404

**排查步骤：**

1. **检查服务状态**
   - 进入云托管控制台
   - 查看服务 `springboot-h9k1` 是否运行中
   - 查看实例数量是否 > 0

2. **查看启动日志**
   - 点击"日志" → "实例日志"
   - 搜索关键字：`Started IdolDiaryApplication`
   - 确认服务已启动

3. **检查接口注册**
   - 在日志中搜索：`Mapped "{[/api/diaries/{id}/pin]"`
   - 确认接口已注册到 Spring MVC

4. **检查流量配置**
   - 确认新版本流量比例 > 0%
   - 如果使用灰度发布，将流量调整到 100%

---

### 问题 4：真机预览仍然失败

**可能原因：**
- 小程序缓存未清除
- Token 过期

**解决：**
1. 在真机预览界面，点击右上角 "..." → 重新编译
2. 退出登录，重新登录获取新 Token
3. 完全关闭小程序，重新打开
