# 日记保存后看不到数据 - 排查指南

## 问题现象
- 保存日记提示"保存成功"
- 但在日记列表看不到新日记

## 可能的原因

### 1. 多爱豆切换问题
**最可能的原因！** 你可能有多个爱豆，新日记保存到了另一个爱豆下。

**排查方法：**
1. 打开调试器 → AppData → app
2. 查看 `globalData.currentIdol.id` （当前选中的爱豆 ID）
3. 查看 `globalData.diaries` 数组
4. 检查每条日记的 `idolId` 字段
5. 如果新日记的 `idolId` 与 `currentIdol.id` 不匹配，说明保存到了其他爱豆

**解决方法：**
- 去"个人中心"切换爱豆
- 或者在写日记时确保选对了爱豆

### 2. 数据未刷新
日记已保存但前端没刷新。

**排查方法：**
1. 保存日记后，手动下拉刷新日记列表
2. 或者切换到其他标签页再切回来
3. 查看 AppData 中 `pages/diary/diary` 的 `diaries` 数组

### 3. 后端保存失败
虽然提示成功，但实际保存失败了。

**排查方法：**
1. 查看后端控制台日志
2. 搜索 `createDiary success - createdId=`
3. 如果没有这行日志，说明保存失败

### 4. 标签数据格式问题
标签数据格式不对导致筛选失败。

**排查方法：**
1. 打开调试器查看 `globalData.diaries`
2. 检查日记的 `tags` 字段是否是数组格式
3. 如果是字符串或其他格式，需要修复

## 快速测试步骤

### 步骤 1：确认当前爱豆
```
调试器 → AppData → app → globalData.currentIdol
记下 id 值，比如 id: 1
```

### 步骤 2：写日记并观察
1. 点击"+"写日记
2. 随便写点内容
3. **选择标签**（点击"演唱会"或"生日"等，选中后会变粉色）
4. 点击"保存"
5. 观察是否跳转回列表页

### 步骤 3：检查数据
```
调试器 → AppData → app → globalData.diaries
查看数组长度是否增加
查看最新一条的 idolId 是否等于 currentIdol.id
查看最新一条的 tags 是否是数组，比如 ["演唱会", "生日"]
```

### 步骤 4：检查页面数据
```
调试器 → AppData → pages/diary/diary
查看 diaries 数组
查看 filteredDiaries 数组
查看 allTags 数组（应该包含你选的标签）
```

## 常见问题

### Q1: 我有多个爱豆怎么办？
**A:** 在写日记前，先去"个人中心"确认当前选中的爱豆。新日记会保存到当前爱豆下。

### Q2: 标签点不了怎么办？
**A:** 
1. 确保你滚动到了标签区域（在页面靠下的位置）
2. 标签现在是白色渐变卡片 + 粉色边框
3. 点击后会变成粉色渐变背景并上浮
4. 如果完全没反应，可能被底部按钮遮挡了

### Q3: 标签筛选栏不显示怎么办？
**A:** 标签筛选栏的显示条件：
1. 必须已登录
2. 必须有爱豆
3. 必须有日记
4. **日记必须有标签**（allTags.length > 0）

如果没有标签，筛选栏不会显示。

### Q4: 为什么我写的日记没有标签？
**A:** 可能的原因：
1. 写日记时忘记选择标签
2. 标签点击没反应（被遮挡或样式问题）
3. 之前保存的日记是没有标签功能时写的

**解决方法：**
1. 编辑已有日记，添加标签
2. 写新日记时记得选择标签

## 调试技巧

### 查看网络请求
1. 打开调试器 → Network 标签
2. 写一条日记并保存
3. 查看 POST /diaries 请求
4. 查看请求数据是否包含 tags 数组
5. 查看响应数据

### 查看控制台日志
1. 打开调试器 → Console 标签
2. 写日记并保存
3. 查看是否有错误信息
4. 特别注意红色的 error 日志

### 强制刷新
如果数据一直不更新：
1. 点击工具栏"清缓存" → "清除全部缓存"
2. 重新编译
3. 重新登录
