# 🔧 解决 JAR 文件被占用问题

## 问题

Maven 构建失败，提示：
```
Unable to rename 'idol-diary-backend-1.0.0.jar' to 'idol-diary-backend-1.0.0.jar.original'
文件正由另一进程使用
```

## 解决方案

### 方案 1：关闭所有占用文件的程序（推荐）

#### 1. 关闭 IDE
- IntelliJ IDEA
- Eclipse
- VS Code
- 或其他打开了该项目的编辑器

#### 2. 关闭文件管理器
- 如果在文件资源管理器中打开了 `backend\target` 文件夹，关闭它

#### 3. 结束 Java 进程
打开任务管理器（Ctrl+Shift+Esc），找到并结束：
- `java.exe` 进程
- `javaw.exe` 进程

#### 4. 重新构建
关闭所有程序后，运行：
```cmd
d:\SOFT\爱豆时光日记\重新构建.cmd
```

---

### 方案 2：使用已编译的 class 文件（快速方案）

如果不想关闭 IDE，可以直接更新现有的 JAR：

#### 步骤 1：复制现有的 JAR
```cmd
copy "d:\SOFT\爱豆时光日记\backend\target\idol-diary-backend-1.0.0.jar" "d:\SOFT\爱豆时光日记\cloud-deploy\app.jar"
```

#### 步骤 2：更新 FileController.class
```cmd
jar -uf "d:\SOFT\爱豆时光日记\cloud-deploy\app.jar" -C "d:\SOFT\爱豆时光日记\backend\target\classes" com/idoldiary/controller/FileController.class
```

这样就把新的 FileController（带测试端点）更新到 JAR 中了。

---

### 方案 3：换一个输出目录

修改构建命令，输出到不同的目录：

```cmd
cd d:\SOFT\爱豆时光日记\backend
mvn clean package -DskipTests -Dmaven.repo.local=C:\temp\m2
```

## 验证 JAR 包含 FileController

构建成功后，验证：

```cmd
jar -tf "d:\SOFT\爱豆时光日记\cloud-deploy\app.jar" | findstr FileController
```

应该看到：
```
BOOT-INF/classes/com/idoldiary/controller/FileController.class
```

## 验证 FileController 有测试端点

查看编译后的 class 文件：

```cmd

```

应该能看到 `test` 方法。

## 最简单的方法

**直接复制 target 目录中已经编译好的 JAR！**

因为我们已经成功编译了代码（FileController.class 已生成），只是最后一步 repackage 失败了。

所以target 目录中的 class 文件已经是最新的了，直接把生成的 JAR 复制到 cloud-deploy 即可：

```cmd
copy "d:\SOFT\爱豆时光日记\backend\target\idol-diary-backend-1.0.0.jar" "d:\SOFT\爱豆时光日记\cloud-deploy\app.jar" /Y
```

如果这个命令也失败（文件被占用），那就关闭所有程序后再试。
