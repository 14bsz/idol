2. 点击顶部 **"云开发"** → **"云托管"**

## ⚠️ 重要提醒

### 必须检查的事项

1. **数据库迁移** ⚠️
   - 必须在部署前执行
   - 否则置顶功能会报错

2. **环境变量** ⚠️
   - 检查云托管服务是否已配置：
     - `MYSQL_ADDRESS`
     - `MYSQL_USERNAME`
     - `MYSQL_PASSWORD`
   - 如果没有，需要在服务设置中添加

3. **部署时间** 💡
   - 建议选择非高峰时段
   - 首次部署需要 10-15 分钟

4. **文件持久化** 💡
   - 容器重启会清空 `uploads` 目录
   - 长期方案：使用对象存储（COS）

## 🆘 遇到问题？

### 常见问题速查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 构建失败 | 网络问题 | 重试部署 |
| 启动失败 | 数据库连接失败 | 检查环境变量 |
| 接口 404 | 旧版本代码 | 确认部署最新 JAR |
| 接口 503 | 服务未启动 | 查看实例状态和日志 |
| 上传失败 | Token 无效 | 重新登录获取 Token |

### 获取帮助

1. **查看实例日志** - 最详细的错误信息
2. **查看构建日志** - Docker 构建过程
3. **检查环境变量** - 数据库连接参数
4. **参考文档** - `cloud-deploy/README.md`
