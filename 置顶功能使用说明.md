# 日记置顶功能使用说明

## ✅ 已完成的功能

### 1. 排序修复

- **修复**：使用时间戳比较，确保最新日记在最上方

- **视觉标识**：置顶的日记会显示金色徽章「📌 置顶」

## 📋 使用前需要执行数据库迁移

### 方式1：通过 Navicat 或其他数据库工具

1. 打开你的数据库管理工具（Navicat、DBeaver、MySQL Workbench 等）
2. 连接到本地数据库：
   - 主机：localhost / 127.0.0.1
   - 端口：3306
   - 用户名：root
   - 密码：123456
   - 数据库：idol_diary

3. 打开并执行文件：`backend/src/main/resources/migration_add_pinned.sql`

4. 执行成功后会看到：
   - diary 表新增了 `pinned` 字段
   - 新增了 `idx_pinned_created_at` 索引

### 方式2：通过命令行（如果配置了 MySQL）

```bash
mysql -u root -p123456 idol_diary < backend/src/main/resources/migration_add_pinned.sql
```

### SQL 内容（可以直接复制执行）

```sql
USE idol_diary;

-- 添加置顶字段
ALTER TABLE `diary` 
ADD COLUMN `pinned` TINYINT NOT NULL DEFAULT 0 COMMENT '是否置顶 0-否 1-是' AFTER `tags`;

-- 添加索引以优化置顶日记的查询
ALTER TABLE `diary` 
ADD INDEX `idx_pinned_created_at` (`pinned`, `created_at` DESC);
```

## 🐛 如果遇到问题

### 问题1：后端报错 Column 'pinned' not found
**原因**：数据库迁移未执行
**解决**：执行上方的 SQL 语句

### 问题2：置顶后没有徽章显示
**原因**：前端代码未更新
**解决**：
1. 微信开发者工具点击「编译」
2. 如果还不行，点击「清缓存」→「清除全部缓存」→「编译」

### 问题3：置顶后日记位置没变
**原因**：可能是排序逻辑问题
**排查**：
1. 打开调试器 → AppData → pages/diary/diary
2. 查看 `filteredDiaries` 数组
3. 检查第一条的 `pinned` 是否为 true
4. 如果不是，检查后端日志

### 问题4：新日记还是显示在下面
**原因**：`createdAt` 字段格式问题
**解决**：查看后端日志，确认 `created_at` 字段的值

## 💡 技术细节

### 前端排序逻辑
```javascript
.sort((a, b) => {
  // 置顶日记优先
  if (a.pinned && !b.pinned) return -1;
  if (!a.pinned && b.pinned) return 1;
  
  // 都置顶或都不置顶时，按创建时间倒序
  const timeA = new Date(a.createdAt || a.createTime).getTime();
  const timeB = new Date(b.createdAt || b.createTime).getTime();
  return timeB - timeA;  // 新的在前
})
```

### 后端接口
- **URL**: `PUT /api/diaries/{id}/pin`
- **请求体**: `{ "pinned": 1 }` 或 `{ "pinned": 0 }`
- **响应**: 返回更新后的日记对象

### 数据库字段
- **字段名**: `pinned`
- **类型**: `TINYINT`
- **值**: 0=普通日记，1=置顶日记
- **默认值**: 0
