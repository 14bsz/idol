# 🚨 404 问题 - 终极排查

## ❌ 问题现象

部署成功，但文件上传接口仍然返回 404：
```json
{
  "timestamp": "2026-06-08T07:17:23.883+00:00",
  "status": 404,
  "error": "Not Found",
  "path": "/api/files/upload"
}
```

## 🔍 可能的原因

### 原因 1：流量还在旧版本（最可能）

**问题：** 新版本部署成功，但流量配置还指向旧版本

**检查：**
1. 云托管控制台 → 版本管理
2. 查看各版本的流量分配
3. 看看 springboot-h9k1-020 是否有流量

### 原因 2：新版本实例未启动

**问题：** 新版本实例启动失败

**检查：**
1. 实例列表
2. 查看 springboot-h9k1-020 实例状态
3. 是否显示"运行中"

### 原因 3：部署的代码不是最新的

**问题：** cloud-deploy.zip 中的 app.jar 是旧的

**检查：**
1. 验证 cloud-deploy/app.jar 的内容
2. 检查是否包含 FileController

---

## ✅ 解决方案

### 方案 1：切换流量到新版本（最可能需要）

#### 步骤 1：打开版本管理

1. 微信开发者工具 → 云托管 → springboot-h9k1
2. 或腾讯云控制台 → 云托管 → springboot-h9k1

#### 步骤 2：查看版本列表

应该看到多个版本：
```
springboot-h9k1-020  ← 新版本（Java 17）流量：0%
springboot-h9k1-019  ← 旧版本（Java 8 失败）
springboot-h9k1-017  ← 更旧版本  流量：100%
...
```

#### 步骤 3：切换流量

**方式 A：直接切换**
1. 找到 springboot-h9k1-020
2. 点击"切换流量"或"设为主版本"
3. 设置流量为 100%
4. 确认

**方式 B：灰度发布**
1. 先设置 springboot-h9k1-020 流量为 10%
2. 测试验证
3. 逐步提升到 50%、100%

#### 步骤 4：验证

重新测试文件上传接口，应该返回 200

---

### 方案 2：停止旧版本

#### 步骤 1：找到旧版本

在版本列表中找到正在运行的旧版本（流量 100% 的那个）

#### 步骤 2：停止旧版本

1. 点击旧版本
2. 选择"停止"或"下线"
3. 确认

#### 步骤 3：确保新版本运行

1. 新版本 springboot-h9k1-020 应该自动接管流量
2. 或手动设置为 100% 流量

---

### 方案 3：删除旧版本并重新发布

如果流量切换不生效：

#### 步骤 1：停止所有旧版本

- springboot-h9k1-017
- springboot-h9k1-019
- 其他旧版本

#### 步骤 2：确保只有新版本运行

只保留 springboot-h9k1-020 运行

#### 步骤 3：测试验证

---

### 方案 4：检查新版本实例日志

#### 步骤 1：找到新版本实例

1. 实例列表
2. 筛选版本：springboot-h9k1-020

#### 步骤 2：查看实例状态

**如果状态是"异常"或"失败"：**
- 查看实例日志
- 搜索 ERROR 或 Exception
- 解决启动问题

**如果没有实例：**
- 说明新版本没有创建实例
- 需要手动创建或重新部署

#### 步骤 3：查看启动日志

搜索：`Started IdolDiaryApplication`

**如果找到：**
- 说明应用启动成功
- 问题可能是流量配置

**如果没找到：**
- 应用启动失败
- 查看错误日志
- 可能是环境变量或其他配置问题

## 🔍 验证 app.jar 内容

### 步骤 1：验证本地 JAR

```cmd
cd d:\SOFT\爱豆时光日记\cloud-deploy

# 检查 JAR 包是否包含 FileController
jar -tf app.jar | findstr FileController
```

**应该看到：**
```
BOOT-INF/classes/com/idoldiary/controller/FileController.class
```

**如果没有：**
- 说明 app.jar 是旧的
- 需要重新复制最新的 JAR

### 步骤 2：重新复制最新 JAR

```cmd
cd d:\SOFT\爱豆时光日记

# 重新构建
cd backend
mvn clean package -DskipTests

# 复制到部署目录
copy target\idol-diary-backend-1.0.0.jar ..\cloud-deploy\app.jar

# 验证
jar -tf ..\cloud-deploy\app.jar | findstr FileController
```

### 步骤 3：重新打包并部署

```cmd
# 删除旧的 ZIP
del cloud-deploy.zip

# 重新打包 cloud-deploy 文件夹为 ZIP
# 使用文件资源管理器或压缩工具

# 重新部署
```
