# 文件上传失败排查指南

## 问题现象
```
[文件上传] 上传失败 
uploadUrl: "http://192.168.111.1:8080/api/files/upload"
Error: 文件上传失败，请检查网络连接
```

## 原因分析

根据日志，上传失败的原因可能是：

1. **后端服务未启动** - Spring Boot 后端没有在 `192.168.111.1:8080` 运行
2. **网络不通** - 测试设备无法访问 `192.168.111.1`
3. **IP 地址变更** - 电脑的 IP 地址已经改变

## 解决方案

### 方案一：使用云托管上传（推荐 ✅）

**适用场景**：真机预览、真机调试、体验版

**优点**：
- 不依赖本地网络
- 部署后可直接使用
- 稳定可靠

**步骤**：

1. **确认云托管已部署并正常运行**
   
   检查文件：`miniprogram/utils/env.js`
   ```javascript
   const UPLOAD_URLS = {
     develop: 'http://192.168.111.1:8080/api/files/upload',
     trial: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload',
     release: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload'
   };
   ```

2. **确认云托管配置正确**
   
   当 `USE_LOCAL_UPLOAD_IN_DEVICE = false` 时，真机预览会自动使用云托管地址

3. **验证云托管文件上传功能**
   
   ```bash
   # 测试云托管上传接口
   curl -X POST https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload \
     -H "Authorization: YOUR_TOKEN" \
     -F "file=@test.jpg"
   ```

### 方案二：修复本地上传（开发调试）

**适用场景**：开发者工具、本地调试

**前提条件**：
- 电脑和手机在同一 WiFi 网络
- 后端服务正在运行
- 防火墙允许 8080 端口访问

**步骤**：

#### 1. 检查电脑 IP 地址

Windows 命令：
```cmd
ipconfig
```

查找 `无线局域网适配器 WLAN` 或 `以太网适配器` 的 IPv4 地址，例如：
```
IPv4 地址 . . . . . . . . . . . . : 192.168.1.100
```

#### 2. 更新配置文件

如果 IP 地址不是 `192.168.111.1`，需要更新 `miniprogram/utils/env.js`：

```javascript
const API_BASE_URLS = {
  develop: 'http://192.168.1.100:8080/api',  // 替换为实际 IP
  trial: '',
  release: ''
};

const UPLOAD_URLS = {
  develop: 'http://192.168.1.100:8080/api/files/upload',  // 替换为实际 IP
  trial: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload',
  release: 'https://springboot-xxpl-267159-8-1440827759.sh.run.tcloudbase.com/api/files/upload'
};
```

#### 3. 确认后端服务运行

```cmd
# 进入后端目录
cd d:\SOFT\爱豆时光日记\backend

# 启动 Spring Boot
mvn spring-boot:run
```

或使用 JAR 包：
```cmd
java -jar target/idol-diary-backend.jar
```

确认看到日志：
```
Tomcat started on port(s): 8080 (http)
```

#### 4. 测试本地连接

在浏览器访问：
```
http://192.168.111.1:8080/api/idols
```

如果无法访问，检查：
- 后端是否正常启动
- 防火墙是否阻止了 8080 端口
- IP 地址是否正确

#### 5. 配置防火墙（Windows）

允许 8080 端口入站：

```cmd
# 以管理员身份运行 PowerShell
netsh advfirewall firewall add rule name="Java Backend 8080" dir=in action=allow protocol=TCP localport=8080
```

#### 6. 启用真机本地上传（可选）

如果希望真机预览也使用本地地址，修改 `miniprogram/utils/env.js`：

```javascript
const USE_LOCAL_UPLOAD_IN_DEVICE = true;  // 改为 true
```

### 方案三：快速切换到云托管（临时方案）

如果本地调试有问题，可以快速切换到云托管模式：

1. 打开微信开发者工具
2. 点击右上角「详情」→「本地设置」
3. 勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」
4. 在代码中临时修改环境判断

修改 `miniprogram/utils/env.js` 的 `useCloudContainer` 函数：

```javascript
const useCloudContainer = (envVersion) => {
  // 临时强制使用云托管
  return true;  // 添加这行，强制所有环境走云托管
  
  const currentEnv = envVersion || getRuntimeEnvVersion();
  if (currentEnv !== 'develop') return true;
  // ... 其他代码
};
```

## 当前日志分析

根据你的日志：

```
[文件上传] 开始上传 {
  uploadUrl: "http://192.168.111.1:8080/api/files/upload",
  useCloud: true,
  envVersion: "develop"
}
```

问题点：
- ✅ `useCloud: true` 表示应该使用云托管
- ❌ 但 `uploadUrl` 仍然是本地地址 `192.168.111.1`
- 🔍 这说明 `getUploadUrl()` 函数逻辑有问题

## 代码修复

问题出在 `getUploadUrl` 函数中的判断逻辑。当前代码：

```javascript
const getUploadUrl = (envVersion) => {
  const currentEnv = envVersion || getRuntimeEnvVersion();
  
  // 真机预览时的上传地址选择
  if (currentEnv === 'develop' && useCloudContainer(envVersion)) {
    if (USE_LOCAL_UPLOAD_IN_DEVICE) {
      return UPLOAD_URLS.develop;
    }
    return UPLOAD_URLS.trial;  // ✅ 应该返回云托管地址
  }
  
  return UPLOAD_URLS[currentEnv] || UPLOAD_URLS.develop;
};
```

当你在真机上测试时：
- `currentEnv = 'develop'`
- `useCloudContainer() = true`（真机无法连本地）
- `USE_LOCAL_UPLOAD_IN_DEVICE = false`
- 应该返回 `UPLOAD_URLS.trial`

**但问题是**：从日志看 `uploadUrl` 仍然是本地地址，说明这个逻辑没有生效。

### 修复方案

更新 `miniprogram/utils/env.js`：

```javascript
const getUploadUrl = (envVersion) => {
  const currentEnv = envVersion || getRuntimeEnvVersion();
  
  // 📱 真机预览时的上传地址选择
  if (currentEnv === 'develop') {
    const isCloudMode = useCloudContainer(envVersion);
    console.log('[env] getUploadUrl - isCloudMode:', isCloudMode, 'USE_LOCAL:', USE_LOCAL_UPLOAD_IN_DEVICE);
    
    if (isCloudMode && !USE_LOCAL_UPLOAD_IN_DEVICE) {
      console.log('[env] 使用云托管上传地址');
      return UPLOAD_URLS.trial;
    }
  }
  
  return UPLOAD_URLS[currentEnv] || UPLOAD_URLS.develop;
};
```

## 常见问题

### Q1: 为什么开发者工具可以上传，真机不行？
A: 开发者工具运行在电脑上，可以访问 localhost，但手机无法访问你电脑的 127.0.0.1 或 192.168.x.x（除非在同一 WiFi）。

### Q2: 如何判断当前使用的是本地还是云托管？
A: 查看控制台日志：
```
[文件上传] 开始上传 { uploadUrl: "...", useCloud: true/false }
```

### Q3: 云托管上传报错怎么办？
A: 
1. 检查云托管服务是否正常运行
2. 检查 Authorization token 是否正确
3. 检查上传文件大小是否超限
4. 查看云托管日志

### Q4: 本地上传报错 "fail:url not in domain list"
A: 在开发者工具勾选「不校验合法域名」，或在微信公众平台添加 request 合法域名。
