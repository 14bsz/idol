# 🚀 正确部署方法 - 本地代码上传

## ❌ 不要使用 GitHub 流水线

**原因：**
- `app.jar` 文件没有提交到 GitHub（被 .gitignore 排除）
- GitHub 流水线克隆代码后找不到 `app.jar`
- 导致构建失败：`"/app.jar": not found`

---

## ✅ 使用本地代码上传

### 步骤 1：切换到「基础信息」标签

在部署配置界面：
1. 点击顶部的 **「基础信息」** 标签
2. 不要使用「流水线」标签

### 步骤 2：选择「本地代码」

在「基础信息」页面：
1. 找到「代码来源」或「上传方式」选项
2. 选择 **「本地代码」** 或 **「本地上传」**
3. 不要选择「代码仓库」或「GitHub」

### 步骤 3：上传 cloud-deploy 文件夹

1. 点击「选择文件夹」或「浏览」
2. 选择文件夹：`d:\SOFT\爱豆时光日记\cloud-deploy\`
3. 或者直接拖拽 `cloud-deploy` 文件夹到上传区域

### 步骤 4：配置部署参数

```
端口：8080
CPU：0.5 核
内存：1 GB
实例数：1
```

### 步骤 5：保存并部署

1. 点击「保存」或「确定」
2. 等待上传完成
3. 点击「部署」
4. 等待 5-10 分钟

## 💡 为什么 app.jar 不在 GitHub？

### .gitignore 配置

```gitignore
# Backend build artifacts
backend/target/
*.jar
*.war
```

**原因：**
- JAR 包是构建产物，通常不提交到版本控制
- 文件太大（32 MB）
- 每次构建都会变化

**但是：**
- 云托管需要这个 JAR 包
- 所以必须用本地上传，不能用 GitHub 流水线

---

## 🔄 备选方案：提交 app.jar 到 GitHub

**如果你坚持用 GitHub 流水线：**

### 1. 修改 .gitignore

临时允许提交这个 JAR：

```gitignore
# Backend build artifacts
backend/target/
*.jar
*.war

# 但允许部署包中的 JAR
!cloud-deploy/app.jar
```

### 2. 提交 app.jar

```cmd
cd d:\SOFT\爱豆时光日记
git add -f cloud-deploy/app.jar
git commit -m "chore: 添加部署 JAR 包"
git push origin main
```

### 3. 重新部署

使用 GitHub 流水线，目标目录：`cloud-deploy`

**缺点：**
- 每次更新代码都要重新提交 32MB 的 JAR 包
- Git 仓库体积会快速增长
- 不推荐

---

## 📊 推荐方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **本地上传** | 简单快速<br>不占用 Git 空间 | 每次都要手动上传 | ⭐⭐⭐⭐⭐ |
| **GitHub + JAR** | 可以用流水线 | Git 仓库变大<br>违反最佳实践 | ⭐⭐ |
| **GitHub + 构建** | 代码和部署分离 | 需要完整源码<br>构建时间长 | ⭐⭐⭐ |
