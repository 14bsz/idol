# 真机调试置顶操作失败问题排查

## 问题现象
真机预览时，置顶操作提示：
```
置顶操作失败: <Error: 请求失败>Error: 请求失败
```
但在微信开发者工具中操作正常。

## 问题原因分析

### 1. 环境判断逻辑
真机预览时，`env.js` 中的 `useCloudContainer()` 会判断为云托管模式：
- 开发者工具：`systemInfo.platform === 'devtools'` → 使用本地 HTTP 直连
- 真机预览：`systemInfo.platform !== 'devtools'` → 使用云托管 `wx.cloud.callContainer`

### 2. 可能的失败原因
1. **云环境未正确初始化** - `wx.cloud.init()` 执行失败或配置错误
2. **云托管服务配置问题** - `CLOUD_SERVICE_NAME` 或 `CLOUD_ENV_ID` 配置错误
3. **网络权限问题** - 真机需要访问云托管服务，可能受网络限制
4. **请求参数序列化问题** - PUT 请求的 data 在云托管中可能序列化方式不同

## 排查步骤

### 步骤 1：查看真机控制台日志
真机预览后，在微信开发者工具中打开"调试器 > Console"查看日志：

1. 确认 App 启动日志：
```
[App] 启动，开始初始化
[App] API配置完成: {...}
[App] 云开发初始化成功, env: prod-d7gv8c42oa25aafb0
```

2. 确认置顶操作日志：
```
[置顶操作] 开始 {...}
[云托管请求] path: /api/diaries/123/pin ...
```

3. 查看失败日志中的具体错误信息

### 步骤 2：检查云托管配置

确认 `env.js` 中的配置：
```javascript
const CLOUD_ENV_ID = 'prod-d7gv8c42oa25aafb0';
const CLOUD_SERVICE_NAME = 'springboot-h9k1';
```

在微信公众平台后台确认：
- 云开发环境 ID 是否正确
- 云托管服务名称是否正确
- 云托管服务是否正常运行

### 步骤 3：检查小程序权限配置

在 `app.json` 或项目设置中确认：
```json
{
  "cloud": true,
  "permission": {
    "scope.userLocation": {
      "desc": "你的位置信息将用于..."
    }
  }
}
```

### 步骤 4：测试云托管连接

创建一个简单的测试页面，直接调用云托管接口：
```javascript
wx.cloud.callContainer({
  config: { env: 'prod-d7gv8c42oa25aafb0' },
  path: '/api/idols',
  method: 'GET',
  success: (res) => {
    console.log('云托管测试成功:', res);
  },
  fail: (err) => {
    console.error('云托管测试失败:', err);
  }
});
```

```javascript

  return false;
```
