# 云托管部署前检查清单

## ✅ 第一步：数据库迁移

### 1.1 检查是否已执行迁移

登录云数据库，执行以下 SQL：

```sql
-- 检查 diary 表是否有 pinned 字段
DESC diary;

-- 检查 collection 表是否有 tags 字段
DESC collection;
```

**预期结果：**
- `diary` 表应该有 `pinned` 字段（INT 类型）
- `collection` 表应该有 `tags` 字段（VARCHAR 类型）

### 1.2 如果字段不存在，执行迁移

**文件：** `云托管数据库完整迁移.sql`

**执行方式 A：云开发控制台**
1. https://console.cloud.tencent.com/tcb
2. 环境：`prod-d7gv8c42oa25aafb0`
3. 数据库 → SQL 编辑器
4. 粘贴脚本，执行

**执行方式 B：MySQL 客户端**
```bash
mysql -h <云数据库地址> -u <用户名> -p < 云托管数据库完整迁移.sql
```

### 1.3 验证迁移结果

```sql
-- 应该看到 pinned 字段
SELECT COLUMN_NAME, DATA_TYPE, COLUMN_DEFAULT 
FROM INFORMATION_SCHEMA.COLUMNS 
WHERE TABLE_NAME = 'diary' AND COLUMN_NAME = 'pinned';

-- 应该看到 tags 字段
SELECT COLUMN_NAME, DATA_TYPE, IS_NULLABLE 
FROM INFORMATION_SCHEMA.COLUMNS 
WHERE TABLE_NAME = 'collection' AND COLUMN_NAME = 'tags';
```

## ✅ 第二步：环境变量检查

### 2.1 登录云托管控制台

1. https://console.cloud.tencent.com/tcb
2. 环境：`prod-d7gv8c42oa25aafb0`
3. 云托管 → 服务 → `springboot-h9k1`

### 2.2 检查环境变量

点击"设置" → "环境变量"，确认以下变量存在：

```
MYSQL_ADDRESS=xxxxxx:3306
MYSQL_USERNAME=root
MYSQL_PASSWORD=xxxxxx
```

### 2.3 如果缺少，添加环境变量

```
变量名：MYSQL_ADDRESS
变量值：你的云数据库地址:3306

变量名：MYSQL_USERNAME
变量值：数据库用户名（通常是 root）

变量名：MYSQL_PASSWORD
变量值：数据库密码
```

**保存后需要重启实例生效**

## ✅ 第三步：部署包检查

### 3.1 确认文件完整

检查 `cloud-deploy` 文件夹包含：

```
cloud-deploy/
├── app.jar          (32 MB，必需)
├── Dockerfile       (必需)
├── README.md        (可选)
└── 部署前检查.md    (本文档，可选)
```

### 3.2 验证 JAR 包

**Windows PowerShell：**
```powershell
# 检查文件大小
Get-Item "d:\SOFT\爱豆时光日记\cloud-deploy\app.jar" | Select-Object Name, Length, LastWriteTime

# 检查 JAR 包内容
jar -tf "d:\SOFT\爱豆时光日记\cloud-deploy\app.jar" | Select-String "FileController"
jar -tf "d:\SOFT\爱豆时光日记\cloud-deploy\app.jar" | Select-String "DiaryService"
```

**预期：**
- 文件大小：约 32 MB
- 包含 FileController.class
- 包含 DiaryService.class

## 🚀 开始部署

检查完成后，按照 `README.md` 中的步骤进行部署：

**推荐方式：微信开发者工具**
1. 打开微信开发者工具
2. 云开发 → 云托管
3. 选择服务 `springboot-h9k1`
4. 上传 `cloud-deploy` 文件夹
5. 配置参数（CPU: 0.5核, 内存: 1GB）
6. 提交部署
7. 等待 5-10 分钟
8. 查看日志确认启动成功

---

## ⚠️ 注意事项

1. **首次部署时间较长**
   - 需要构建 Docker 镜像
   - 预计 5-15 分钟

2. **确保数据库可访问**
   - 云托管实例必须能连接到云数据库
   - 检查网络配置和安全组

3. **文件上传路径**
   - 容器内路径：`/app/uploads`
   - 容器重启后文件可能丢失
   - 长期方案：使用对象存储（COS）

4. **实例健康检查**
   - 端口：8080
   - 健康检查路径：可设置为 `/api/idols`（需要 Token）
   - 或设置为自定义健康检查端点

## 📞 遇到问题？

1. **查看部署日志**
   - 云托管控制台 → 构建日志
   - 查看具体错误信息

2. **查看实例日志**
   - 云托管控制台 → 实例日志
   - 搜索 "ERROR" 或 "Exception"

3. **常见错误**
   - "数据库连接失败" → 检查环境变量
   - "启动超时" → 增加内存配置
   - "端口被占用" → 检查端口配置

4. **回滚**
   - 版本管理 → 选择上一版本 → 回滚
